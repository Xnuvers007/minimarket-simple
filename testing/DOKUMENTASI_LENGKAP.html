1. Pendahuluan

Apa itu Sistem Minimarket Sejahtera?

Sistem Informasi Minimarket Sejahtera adalah aplikasi web modern untuk mengelola operasional toko minimarket dengan fitur lengkap meliputi:

  • Manajemen produk dan kategori
  • Sistem Point of Sale (POS) untuk kasir
  • Platform belanja online untuk customer
  • Manajemen stok dan inventory
  • Laporan penjualan dan transaksi
  • Manajemen pengguna dan role
‚úÖ Teknologi yang Digunakan:
  • PHP 8.0+ (Backend)
  • MySQL 5.7+ (Database)
  • HTML5, CSS3, JavaScript (Frontend)
  • Bootstrap 5 (UI Framework)
  • Chart.js (Analytics)

2. Fitur Utama

Fitur yang Tersedia

üì¶ Manajemen Produk

Tambah, edit, hapus produk dengan upload foto, kategori, harga, dan stok.

üõí Point of Sale (POS)

Kasir dapat memproses transaksi cepat dengan cart realtime dan berbagai metode pembayaran.

üõçÔ∏è Toko Online

üìä Laporan & Analitik

4. Panduan Instalasi

Prasyarat

  • PHP 8.0 atau lebih baru
  • MySQL 5.7 atau lebih baru
  • Web Server (Apache/Nginx)
  • Browser modern (Chrome, Firefox, Safari)

Langkah Instalasi

1 Download File Aplikasi

Download semua file aplikasi dari repository atau folder project.

2 Copy ke Web Root

Copy folder sera ke folder htdocs (XAMPP) atau www (WAMP).

XAMPP: C:\xampp\htdocs\sera\
WAMP: C:\wamp64\www\sera\
3 Buat Database

Buka phpMyAdmin dan buat database baru bernama minimarket_db

4 Import Database

Import file database.sql ke database minimarket_db

phpMyAdmin ‚Üí Import ‚Üí pilih database.sql
5 Konfigurasi Database (Optional)

Edit config.php jika username/password MySQL berbeda:

define('DB_HOST', 'localhost');

define('DB_NAME', 'minimarket_db');
6 Akses Aplikasi

11. Troubleshooting

Login Gagal

Problem: Username/Password salah atau akun tidak ditemukan
Solusi:
  • Pastikan username dan password benar (case-sensitive)
  • Cek CAPS LOCK tidak aktif
  • Reset password melalui admin jika lupa
  • Pastikan database sudah diimport

Database Tidak Terhubung

Problem: Error "Connection failed" atau "Database error"
Solusi:
  • Pastikan MySQL/MariaDB sudah running
  • Cek config.php untuk DB_HOST, DB_USER, DB_PASS benar
  • Pastikan database minimarket_db sudah dibuat
  • Restart web server (Apache/Nginx)

Halaman Blank/Error 500

Problem: Halaman kosong atau error 500
Solusi:
  • Cek file permission (755 untuk folder, 644 untuk file)
  • Lihat PHP error log di console atau terminal
  • Set error_reporting di config.php
  • Cek syntax PHP (gunakan `php -l filename.php`)

File Upload Gagal

Problem: Foto produk tidak bisa diupload
Solusi:
  • Pastikan folder assets/images/products/ writable (chmod 777)
  • Cek ukuran file tidak melebihi 5MB
  • Pastikan format file JPG, PNG, atau GIF
  • Restart web server

POS Cart Tidak Sync

Problem: Barang tidak masuk keranjang POS
Solusi:
  • Reload halaman pos.php
  • Clear browser cache (Ctrl+Shift+Del)
  • Buka browser console (F12) cek error JavaScript
  • Cek session PHP aktif

Transaksi Tidak Tersimpan

Problem: Checkout gagal atau transaksi tidak masuk database
Solusi:
  • Pastikan cart tidak kosong sebelum checkout
  • Cek jumlah bayar ‚â• grand total
  • Lihat MySQL error log untuk detail error
  • Cek foreign key constraints di database

Laporan Tidak Muncul Data

Problem: Laporan kosong atau tidak ada data
Solusi:
  • Cek filter tanggal tidak terlalu jauh ke masa lalu
  • Pastikan ada data transaksi di database
  • Buka browser console cek error ajax
  • Cek query SQL di admin/reports.php

Performance Lambat

Problem: Aplikasi berjalan sangat lambat
Solusi:
  • Optimize database dengan indexing tabel besar
  • Limit hasil query (LIMIT 1000)
  • Enable query caching di MySQL
  • Upgrade PHP version ke 8.1+
  • Upgrade RAM server
üí° Bantuan Lebih Lanjut:
Jika masalah tidak teratasi, hubungi tim support atau cek error log untuk detail lengkap.

11. Troubleshooting

Login Gagal

Problem: Username/Password salah atau akun tidak ditemukan
Solusi:
  • Pastikan username dan password benar (case-sensitive)
  • Cek CAPS LOCK tidak aktif
  • Reset password melalui admin jika lupa
  • Pastikan database sudah diimport

Database Tidak Terhubung

Problem: Error "Connection failed" atau "Database error"
Solusi:
  • Pastikan MySQL/MariaDB sudah running
  • Cek config.php untuk DB_HOST, DB_USER, DB_PASS benar
  • Pastikan database minimarket_db sudah dibuat
  • Restart web server (Apache/Nginx)

Halaman Blank/Error 500

Problem: Halaman kosong atau error 500
Solusi:
  • Cek file permission (755 untuk folder, 644 untuk file)
  • Lihat PHP error log di console atau terminal
  • Set error_reporting di config.php
  • Cek syntax PHP (gunakan `php -l filename.php`)

File Upload Gagal

Problem: Foto produk tidak bisa diupload
Solusi:
  • Pastikan folder assets/images/products/ writable (chmod 777)
  • Cek ukuran file tidak melebihi 5MB
  • Pastikan format file JPG, PNG, atau GIF
  • Restart web server

POS Cart Tidak Sync

Problem: Barang tidak masuk keranjang POS
Solusi:
  • Reload halaman pos.php
  • Clear browser cache (Ctrl+Shift+Del)
  • Buka browser console (F12) cek error JavaScript
  • Cek session PHP aktif

Transaksi Tidak Tersimpan

Problem: Checkout gagal atau transaksi tidak masuk database
Solusi:
  • Pastikan cart tidak kosong sebelum checkout
  • Cek jumlah bayar ‚â• grand total
  • Lihat MySQL error log untuk detail error
  • Cek foreign key constraints di database

Laporan Tidak Muncul Data

Problem: Laporan kosong atau tidak ada data
Solusi:
  • Cek filter tanggal tidak terlalu jauh ke masa lalu
  • Pastikan ada data transaksi di database
  • Buka browser console cek error ajax
  • Cek query SQL di admin/reports.php

Performance Lambat

Problem: Aplikasi berjalan sangat lambat
Solusi:
  • Optimize database dengan indexing tabel besar
  • Limit hasil query (LIMIT 1000)
  • Enable query caching di MySQL
  • Upgrade PHP version ke 8.1+
  • Upgrade RAM server
üí° Bantuan Lebih Lanjut:
Jika masalah tidak teratasi, hubungi tim support atau cek error log untuk detail lengkap.
