==========================================================
  TROUBLESHOOTING XAMPP
==========================================================

âŒ Problem: Apache tidak bisa Start (port 80 sudah digunakan)
   Gejala: Background Apache tetap putih/merah, error "Port 80 in use"
   
   Penyebab: Ada aplikasi lain yang pakai port 80 (Skype, IIS, dll)
   
   Solusi 1 - Ganti Port Apache:
   1. Di XAMPP Control Panel, klik tombol "Config" di sebelah Apache
   2. Pilih "httpd.conf"
   3. Tekan Ctrl+F, cari: Listen 80
   4. Ganti jadi: Listen 8080
   5. Cari lagi: ServerName localhost:80
   6. Ganti jadi: ServerName localhost:8080
   7. Save file (Ctrl+S)
   8. Tutup editor
   9. Klik "Stop" lalu "Start" Apache lagi
   10. Sekarang buka: http://localhost:8080/sera/
   
   Solusi 2 - Matikan Aplikasi yang Konflik:
   1. Tekan Ctrl+Alt+Delete
   2. Pilih "Task Manager"
   3. Cari aplikasi yang pakai port 80:
      - Skype â†’ Klik kanan â†’ End Task
      - IIS (World Wide Web Publishing) â†’ End Task
   4. Start Apache lagi

âŒ Problem: MySQL tidak bisa Start (port 3306 sudah digunakan)
   Gejala: Background MySQL tetap putih/merah
   
   Penyebab: Ada MySQL lain yang sudah running
   
   Solusi:
   1. Tekan Windows+R
   2. Ketik: services.msc
   3. Enter
   4. Cari service bernama "MySQL" atau "MySQL80"
   5. Klik kanan â†’ Stop
   6. Start MySQL di XAMPP lagi

âŒ Problem: "Access Denied" saat buka http://localhost/sera/
   Solusi:
   1. Pastikan folder "sera" ada di C:\xampp\htdocs\
   2. Bukan di folder lain!
   3. Path yang benar: C:\xampp\htdocs\sera\index.php

âŒ Problem: "Table doesn't exist" saat login
   Solusi:
   1. Database belum diimport
   2. Ulangi LANGKAH 4 untuk import database.sql
   3. Pastikan semua 11 tabel ada di phpMyAdmin

âŒ Problem: "Upload failed" saat upload gambar produk
   Solusi:
   1. Buka: C:\xampp\htdocs\sera\assets\images\products\
   2. Klik kanan folder "products" â†’ Properties
   3. Hilangkan centang "Read-only"
   4. Klik "Apply to all files and folders"
   5. OK
   6. Coba upload lagi

âŒ Problem: Halaman blank putih setelah login
   Solusi:
   1. Tekan F12 (Chrome DevTools)
   2. Klik tab "Console"
   3. Lihat error message
   4. Biasanya masalah path atau config.php
   5. Screenshot error dan cek DOKUMENTASI_LENGKAP.md
